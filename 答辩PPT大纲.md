## 第3页：通信与物联网

**标题：** 串口通信 + ESP8266 + MQTT + 华为云

**左上——三路 USART（小表格）：**
| USART | 引脚 | 波特率 | 用途 |
|-------|------|--------|------|
| USART1 | PA9/PA10 | 9600 | 调试串口（上位机） |
| USART2 | PA2/PA3 | 115200 | HC-05 蓝牙（手机） |
| USART3 | PB10/PB11 | 115200 | ESP8266 WiFi 模块 |

**右上——ESP8266 连接流程（流程图）：**
```
AT+CWJAP → WiFi 连接（超时10s×3次重试）
    ↓
AT+CIPSTART → TCP 连接（华为云 1883）
    ↓
AT+CIPMODE=1 + AT+CIPSEND → 进入透传
    ↓
MQTT CONNECT → 等待 CONNACK
    ↓
SUBSCRIBE → 订阅消息/命令 Topic
```

**左下——MQTT 报文（代码框）：**
```
CONNECT: 0x10 + 剩余长度 + "MQTT" + 4 + 0xC2 + KeepAlive 60s + ClientID/Username/Password
PUBLISH: 0x30 + 剩余长度 + Topic + Payload（QoS 0）
PINGREQ: 0xC0 0x00（每30秒）

Topic 设计：
  $oc/devices/{id}/sys/properties/report       → 上报温湿度
  $oc/devices/{id}/sys/messages/down            ← 订阅消息
  $oc/devices/{id}/sys/commands/#               ← 订阅命令
  $oc/devices/{id}/sys/commands/response/...    → 命令响应
```

## 第4页：核心算法

**标题：** 空间扫描 + 自动避障 + 自动跟随

**左上——空间扫描（图+文）：**
```
9扇区 180° 扫描（30°~150°，步进15°）
角度查表：{30,45,60,75,90,105,120,135,150}

最长连续可通行段决策：
  g_map[i] > threshold → 可通行
  找最长连续可通行段
  覆盖正中央(索引4) → 直行
  中心偏左 → 左转，偏右 → 右转
  全部不通 → 脱困
```

**右上——避障状态机（转移图）：**
```
   ┌──────┐  红外触发/距离<阈值   ┌──────┐
   │ 前进  │ ──────────────────→ │ 扫描  │
   │(状态0)│                     │(状态1)│
   └──┬───┘                     └──┬───┘
      ↑                           │
      │    ┌──────┐  有通路        │
      └────│ 掉头  │←──────┐      │
           │(状态3)│       │      │
           └───────┘   ┌──┴───┐  │
                       │试探转向│←┘
                       │(状态2)│ 不通→掉头
                       └──────┘

红外优先级最高：左右同时障碍→后退500ms→左转400ms
```

**底部——跟随模式（文）：**
```
舵机固定 90° 正前方测距
  d ≥ 10cm 且 d ≤ 15cm → Motor_Forward(speed)   正常表情
  d < 10cm             → Motor_Backward(speed×3/4) 警戒表情
  d > 15cm 或 d ≤ 0    → Motor_Stop()             停车等待
```
